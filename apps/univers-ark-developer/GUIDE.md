## 2. 主页 Dashboard

打开应用后进入主页，显示整个工作区的概况。

### 信息卡片

**顶部摘要栏**
- 已配置的机器数量
- AI Agent 团队数量
- SSH 可用的容器数量
- **Add Machine** 按钮：添加新机器
- **Open Overview** 按钮：跳转到终端总览

**Machines（机器列表）**
每台机器显示：主机名、容器数量、SSH 就绪数量、连接状态。
点击机器名进入该机器的详情页；点击齿轮图标编辑机器配置。

**Agent Teams（容器列表）**
所有容器的汇总视图，显示：所属机器、IP 地址、主要 Web 服务状态、SSH 连接状态。
点击 **Open** 直接进入容器工作区。

**Standalone Targets（独立目标）**
不属于任何托管机器的单独目标（如本地 `localhost`），同样可点击 **Open** 打开。

## 3. 总览页 Overview

点击侧边栏 **Overview** 进入，以**终端卡片网格**的形式展示所有容器。

### 功能

- **实时终端预览**：每张卡片都是一个缩小的实时终端，滚动出视口时自动暂停以节省资源
- **点击打开工作区**：点击卡片右上角的展开图标，进入该容器的全屏工作区
- **缩放**：状态栏右侧的 `−` / `+` 按钮调整卡片大小，`○` 重置默认缩放
- **刷新**：点击右上角刷新按钮重新扫描所有机器的容器状态
- **键盘导航**：方向键在卡片间移动，`Enter` 打开工作区

### 状态指示

- 卡片正常显示 → SSH 连接正常
- 卡片显示"Unavailable"及重试按钮 → 该容器 SSH 不可达

## 4. 机器管理

### 添加机器

1. 主页点击 **Add Machine** 或设置页点击 **+ Add**
2. 选择导入方式：
   - **手动填写**：输入主机名、SSH 用户、端口、私钥路径
   - **从 SSH Config 导入**：自动读取 `~/.ssh/config` 中的主机别名
   - **从 Tailscale 导入**：扫描 Tailscale 网络中的对等节点
3. 保存后应用自动尝试连接并发现容器

### 机器详情页

点击侧边栏中的机器名进入，包含两个标签：

**Dashboard 标签**
- 顶部摘要：服务器主机名、容器总数、SSH 就绪数、整体状态
- 容器列表：每个容器的名称、IP、SSH 状态，以及 **Open** 按钮直接进入工作区

**Terminals 标签**
- 左侧：机器宿主机的 Shell 终端（直接 SSH 到服务器）
- 右侧：该机器上所有容器的终端卡片网格

## 5. 容器工作区

点击任意容器进入工作区，界面分为**左侧终端**和**右侧工具面板**两部分，可拖动中间分割线调整比例。

右侧工具面板有四个标签（通过右上角图标切换）：

| 图标 | 标签 | 说明 |
|------|------|------|
| 方格 | Dashboard | 容器运行状态总览 |
| 列表 | Services | 服务管理 |
| 文件夹 | Files | 文件浏览 |
| 地球 | Browser | 内嵌浏览器 |

顶部还有两个控制按钮：
- **终端折叠/展开**（左上角图标）：隐藏终端面板，让工具面板占满全宽
- **重启容器**（循环箭头图标，仅托管容器可用）

### 5.1 终端

直接连接到容器的 Shell，功能与普通终端一致。

- 点击终端区域自动获得焦点
- 断连时点击右上角**重连按钮**（循环箭头）重建连接
- 状态指示：`Connecting` → `Connected` → 正常使用

### 5.2 Dashboard 标签

自动定期刷新（可在设置中配置间隔），显示以下信息：

**顶部摘要栏**
- 项目根路径
- 主要 Web 服务 URL
- 上次刷新时间
- 手动刷新按钮（循环箭头）及当前刷新间隔标签

**Git 卡片**
| 字段 | 说明 |
|------|------|
| Repo | 项目仓库是否存在（ready / missing） |
| Branch | 当前 Git 分支名 |
| Working tree | 工作区是否干净（clean / dirty），以及变更文件数 |

**Service 卡片**
显示容器内运行的各项服务状态（running / down / healthy / unknown），以及服务 URL。

**Container 卡片**
主机名、运行时长、内存/磁盘用量（已用 / 总量 / 百分比）、进程数、1m/5m/15m 负载均值。

**Agent 卡片**
当前活跃的 AI Agent（claude / codex 等）、来源、最后活动时间、最新报告摘要。

**Tmux 卡片**
tmux 是否已安装、服务器是否在运行、所有会话列表（会话名、窗口数、活跃命令、是否已附加）。右上角 **Restart** 按钮可重启 tmux 服务器。

### 5.3 Services 标签

按类型分组展示容器的所有服务，支持直接操作。

**Web 服务**
- 显示服务名称、状态徽章、描述、本地 URL
- **View** 按钮：在右侧 Browser 标签中打开
- **Open** 按钮：在系统默认浏览器中打开

**Endpoint 服务**
- 显示 HTTP/TCP 探测类型、主机、端口
- 显示当前可访问的 URL

**Command 服务**
- 显示命令内容
- **Restart** 按钮：在容器内执行对应命令（用于重启开发服务器等）

### 5.4 Files 标签

浏览容器（或本地）文件系统，支持多种视图。

**工具栏操作**

| 控件 | 说明 |
|------|------|
| ↑ 上一级 | 导航到父目录 |
| 根目录下拉框 | 快速跳转到常用目录（home、项目目录等） |
| 当前路径 | 显示当前所在目录的完整路径 |
| Icons / Details | 切换图标视图和详情列表视图 |
| 隐藏文件切换 | 显示或隐藏以 `.` 开头的文件 |
| 刷新 | 重新加载当前目录 |

**文件操作**
- 点击文件夹 → 进入该目录
- 点击文件 → 在右侧预览面板打开文件内容（纯文本预览，支持代码高亮）
- 详情视图中点击列标题 → 按名称/大小/修改时间排序

**支持的目标类型**
- SSH 容器：通过 SSH 连接浏览远端文件系统
- Docker 容器：通过 `docker exec` 浏览容器内文件
- 本地（localhost）：直接浏览本地文件系统

### 5.5 Browser 标签

内嵌 Web 浏览器，直接在应用内预览 Web 服务。

**顶部工具栏**

| 控件 | 说明 |
|------|------|
| 服务下拉框 | 切换不同 Web 服务（含动态发现的本地服务） |
| 全屏按钮 | 让浏览器面板占满整个工具区，按 `Esc` 退出 |
| 状态点 / 徽章 | 隧道连接状态（Direct / Running / Starting / Error） |
| Reload | 刷新 iframe 内容 |
| Open | 在系统默认浏览器中打开当前 URL |

**服务下拉框说明**

下拉框中会自动列出两类服务：
1. **配置中的服务**：`developer-targets.json` 中定义的 Web 服务
2. **动态发现的服务**：应用每 10 秒扫描本地监听端口，自动探测 HTTP 服务（显示页面标题或端口号作为标签），无需手动配置

选择不同服务后，下方内容区立即切换到对应 URL。

## 6. Docker 容器

### 添加 Docker 容器

1. 确保 **Docker Desktop 已启动**（系统托盘中可见 Docker 图标）
2. 点击侧边栏底部 **+** → **Add Docker Container**
3. 从检测到的本地容器列表中勾选目标容器（非 SSH 端口的映射端口会一并显示）
4. 确认后容器出现在侧边栏 **Docker (local)** 分组下

### Docker 服务器页

点击侧边栏中 **Docker (local)** 进入，显示：

- **顶部摘要**：容器总数、运行中数量、服务器状态
- **Containers 卡片**：列出每个容器并实时刷新（每 5 秒）以下指标：
  - 状态（running / exited / paused）
  - CPU 使用率
  - 内存用量及百分比
  - 网络 I/O
  - PID 数量
  - **Terminal** 按钮：打开该容器的工作区

## 7. 设置

点击状态栏左下角**齿轮图标**进入设置页，包含四个标签：

### Appearance（外观）

| 设置项 | 选项 |
|--------|------|
| 主题 | System（跟随系统） / Light / Dark |
| Dashboard 刷新间隔 | Off（手动） / 15s / 30s / 60s / 5m |

### Configuration（配置文件）

- 显示当前 `developer-targets.json` 的文件路径
- 内嵌 JSON 编辑器，直接编辑后点击 **Save** 保存并重新加载配置

### Machines（机器管理）

- 查看所有已配置机器的连接状态
- 点击机器名编辑其配置（主机、端口、SSH 用户、私钥、跳板机等）
- **+ Add** 按钮添加新机器

### Profiles（配置档案）

- 查看和管理配置档案（Profiles），档案定义了容器的默认工作区行为（默认工具、项目路径、服务列表等）
- 设置默认档案
- **+ Add** 添加新档案

## 8. 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Alt+Left` | 切换到上一个容器 |
| `Ctrl+Alt+Right` | 切换到下一个容器 |
| `Ctrl+Alt+Up` | 返回上级视图（容器 → 机器 → 主页） |
| `Esc` | 退出浏览器全屏模式 |
| `↑ ↓ ← →` | 在 Overview 卡片间导航 |
| `Enter` | 在 Overview 中打开当前选中的容器工作区 |

> macOS 上 `Ctrl` 替换为 `Cmd`。
